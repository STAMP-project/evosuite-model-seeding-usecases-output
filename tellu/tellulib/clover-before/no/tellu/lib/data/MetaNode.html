4    /**
5    * MetaNode objects specify properties and requirements
6    * on DataNodes, such as data type, cardinality and
7    * validation. MetaNode is the abstract super-class for
8    * MetaNode classes, enabling a class hierarchy and a tree
9    * structure of MetaNode objects. Each node has a name,
10    * to match the name of DataNodes the MetaNode pertains to.
11    * Cardinality is also a general property of MetaNodes,
12    * which can be applied to both value nodes and internal
13    * nodes. And MetaNode can flag nodes as readOnly,
14    * meaning external code is not allowed to modify their
15    * content.
16    * <p>
17    * MetaNode requirements work on two levels. Some are
18    * absolute requirements which DataNodes must abide by,
19    * such as not allowing multiple nodes where cardinality
20    * specifies (at most) one, and not allowing data of
21    * incompatible values. Others are used for validation.
22    * Validity is a DataNode property, so it is possible for
23    * DataNode content to be invalid, such as having no
24    * node or value where cardinality specify at least one.
25    * It is up to the client code to decide if invalid
26    * content is tolerated. A good example is an input form,
27    * where some fields start out empty but input is required
28    * before the form can be submitted.
29    * <p>
30    * MetaNodes will typically form a tree structure
31    * corresponding to the DataNode tree, with the exception
32    * that there is only one MetaNode for a set of child
33    * DataNodes with the same name (these are instances of
34    * the same meta). MetaNode is immutable.
35    *
36    * @author Lars Thomas Boye, Tellu AS
37    */
