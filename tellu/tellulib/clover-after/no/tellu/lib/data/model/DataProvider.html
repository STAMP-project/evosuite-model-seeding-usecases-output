15    /**
16    * A DataProvider is an active provider of data stored in a
17    * DataModel, responsible for keeping up-to-date values in the
18    * model as long as there are subscribers interested in this
19    * data. The framework keeps track of activations and potentially
20    * configurations for individual activations, as well as the
21    * connection to the model. To implement a DataProvider, there
22    * are some abstract protected methods to implement, which are
23    * called by the framework to get some attributes and start and
24    * stop the provider. And the superclass has protected
25    * transaction methods to call for providing data. While active,
26    * the implementation must use the transaction methods to put
27    * values into the model each time it detects a change. A
28    * typical DataProvider is a listener for some form of event,
29    * mapping this into DataModel values.
30    * <p>
31    * A DataProvider has a name which is used to uniquely identify
32    * it in a model. All data provided is put into a DataGroupNode
33    * with the same name as the provider. The location of this group
34    * within the data tree of the model is specified when adding the
35    * provider to a model. The group and its contents could well be
36    * defined as read-only by meta-data, to prevent other code from
37    * modifying the values, as the DataProvider transaction will
38    * still be able to write to this group.
39    *
40    * @author Lars Thomas Boye, Tellu AS
41    */
