14    /**
15    * The DataTemplate provides instances of a hierarchical data
16    * structure based on a template structure. The template structure
17    * of DataNodes can have placeholder values, which specify
18    * Substituters used to generate values for each instance. The
19    * rest of the template is unchanged. DataTemplate is a
20    * DataHandle, which generates an instance on construction and
21    * on each call of nextNode. The handle can be used as normal to
22    * access and modify the current instance. You typically either
23    * want to get the instance with getNode, or serialize it. Once
24    * you call nextNode, the DataTemplate no longer holds a
25    * reference to the previous instance. Transaction and subscription
26    * handling is not implemented here.
27    * <p>
28    * Substituters are specified in the format
29    * <code>$SUBNAME:ARGUMENT1:ARGUMENT2:...</code>. SUBNAME is the
30    * name of the Substituter. Each named substituter has a
31    * corresponding class inheriting from abstract class Substituter,
32    * and the mapping is made by the substituterFactory method of
33    * DataTemplate. A number of Substituters are provided, and the
34    * factory method can be overridden if custom Substituters are
35    * needed. Depending on the type, a Substituter may have one or
36    * more arguments, separated by colons.
37    * <p>
38    * Substitution can iterate over a separate data structure
39    * providing values, with DATA substituters specifying paths in
40    * this data source. The template is constructed with the data
41    * source and a path for iteration. The path can have multiple
42    * levels. With a path "A/B" it will iterate over every B in
43    * every A. See MultiLevelHandle for more on multi-level iteration.
44    * if the template has a data source, it provides as many
45    * instances as there are nodes to iterate over in the source,
46    * nextNode eventually returning false. Call resetNode to reset
47    * dataSource iteration. If there is no data source, there is no
48    * limit on the number of instances. The resetSubstituters method
49    * resets substituters as well as the data source, so is a more
50    * complete reset.
51    *
52    * @author Lars Thomas Boye, Tellu
53    */
