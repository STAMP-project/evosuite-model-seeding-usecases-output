6    /**
7    * MultiLevelHandle provides multi-level iteration over nodes.
8    * Let's say that under a root node R there are multiple A-nodes,
9    * and under each A there are multiple B-nodes. We make a
10    * MultiLevelHandle for the root, with subHandlePath "A/B".
11    * Initially, if we ask for "A" we get the first A and if we ask
12    * for "A/B" we get the first B under the first A. But if we call
13    * nextNode and do the same, we still get the first A if we ask
14    * for "A", but the second B under this A if we ask for "A/B".
15    * Once we have iterated over all B nodes under the first A, the
16    * next call to nextNode skips to the second A node and the first
17    * B under this. So if there are a total of 10 B nodes under the
18    * A nodes, nextNode can be called 10 times before returning
19    * false to signal the end of the iteration. If there is an A
20    * node with no B nodes, it is skipped. The class is used
21    * recursively to support any number of levels, and it is the
22    * last level in the subHandlePath which controls the iteration.
23    * Note that through this handle you will only see a single A
24    * with a single B at any one time, and predicates for the
25    * subHandlePath is ignored, so "A[2]/B[3]" will find whatever A
26    * and B is currently the context. But every level of the handle
27    * can have other nodes not iterated over, and these are accessed
28    * as normal. Once all B nodes have been iterated over, the
29    * handle has no context node (hasNode() is false). resetNode()
30    * will reset the state, with the first B of the first A as
31    * context. totalNodes() returns the number of B nodes.
32    * <p>
33    * One important restriction of this handle is that it is
34    * read-only. Attempting to call setValue or any of the other
35    * modifying methods will throw a TransactionException.
36    * Transaction and subscription handling is naturally not
37    * supported. But other methods work as expected, and it is
38    * possible to get other handles or direct node access, but any
39    * change to the nodes while they are accessed with a
40    * MultiLevelHandle should be done with care.
41    *
42    * @author Lars Thomas Boye, Tellu AS
43    */
