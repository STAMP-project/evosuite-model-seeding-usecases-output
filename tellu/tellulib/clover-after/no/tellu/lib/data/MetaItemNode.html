6    /**
7    * A MetaItemNode specifies properties/requirements and
8    * processing on DataValueNodes. It specifies type and
9    * requirements for validation. A default value is also
10    * defined, to be used initially or when values are reset.
11    * Cardinality has an additional meaning for MetaItemNode:
12    * if a node is required, it also must have a non-null
13    * value to be valid. Default cardinality of items is
14    * CARD_ZERO_OR_ONE, which means a null value is allowed.
15    * A different cardinality can be specified with the full
16    * constructor - CARD_ONE to require a non-null value,
17    * and CARD_ZERO_OR_MORE or CARD_ONE_OR_MORE to allow
18    * multiple values (array). MetaItemNode is immutable,
19    * except for an optional DataValidator object to provide
20    * additional validation.
21    *
