13    /**
14    * The Rule class implements Trigger-Condition-Action rules in
15    * DataModel. A Rule applies to any number of DataGroupNodes based
16    * on a context node path, and optional predicate to select a
17    * subset of the nodes matching the path. It creates some state
18    * data under each context node - a boolean condition value for
19    * each condition in a condition group, and a boolean rule value
20    * in a rule group.
21    * <p>
22    * A Rule can have a list of Conditions - components which update
23    * a boolean condition value based on whether a condition is met
24    * or not. A condition component can be parsed from a boolean
25    * expression. A Rule can have a list of Triggers - components
26    * which call a trigger method on some event, to trigger the rule
27    * with a predicate identifying a context node. If the rule has
28    * only conditions, it will fire when all conditions are true. If
29    * it has only triggers, it will fire on any trigger. If it has
30    * both, it will fire on any trigger, if all conditions are true.
31    * Finally a Rule has a list of Actions - components executed
32    * when the rule fires.
33    *
34    * @author Lars Thomas Boye, Tellu AS
35    */
